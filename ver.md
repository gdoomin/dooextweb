# Version Rule

- For every change, bump the version by `0.0.1` before `commit / push / deploy`.
- After `4.1.9`, the next version is `4.2.0` (no `4.1.x` beyond 4.1.9).
- Keep the latest value in this file.
- Before editing any code, verify Korean text safety first:
  - Use UTF-8 editing only.
  - Do not overwrite Korean-containing files with unspecified encoding.
  - If mojibake appears, stop and restore encoding before any feature work.
